# samples

`samples`는 Goondan 기능을 실행 가능한 번들로 검증하는 레퍼런스 영역이다.

## 존재 이유

- 사용자 온보딩과 회귀 검증에 모두 활용 가능한 실제 시나리오를 제공한다.
- 스펙/코어 변경이 사용자 체감 동작에 미치는 영향을 빠르게 확인한다.

## 구조적 결정

1. 샘플은 독립 실행 가능한 번들 단위로 유지한다.
이유: 특정 샘플 변경이 다른 샘플 실행성을 깨지 않게 하기 위해.
2. 샘플 계약은 `README`와 프롬프트에 명시한다.
이유: 실행 기대 동작을 코드 밖에서도 재현 가능하게 하기 위해.
3. 로컬 상대경로보다 패키지 의존 ref를 우선한다.
이유: 환경 독립성과 배포 재현성을 높이기 위해.

## 불변 규칙

- 샘플 추가/변경 시 루트 안내 문서의 샘플 목록 동기화를 검토한다.
- self-evolution 샘플은 재기동 트리거 조건과 호출 위치를 명시적으로 유지한다.
- 런타임 정보가 필요하면 암묵 주입보다 명시 API 사용 기준을 유지한다.
